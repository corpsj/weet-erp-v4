## 1. 시스템 개요

WEET 마케팅 자동화는 **AI 기반 마케팅 데이터 수집 → 분석 → 제안 → 콘텐츠 생성**을 자동으로 수행하는 시스템입니다.

### 구성 요소

| 구성 요소 | 역할 | 접속 방법 |
|-----------|------|----------|
| **ERP 대시보드** (Next.js) | 데이터 시각화, 제안 승인/거부, 리드 관리 | `https://erp.we-et.com` → 로그인 → 사이드바 "마케팅" |
| **Python 스케줄러** (weet-director) | 시장 조사, 리드 수집, AI 제안/콘텐츠 생성 | 로컬 서버에서 실행 |
| **Supabase** | 데이터 저장소 (클라우드 DB) | 자동 연결 (설정 불필요) |
| **Ollama** | 로컬 AI 모델 (콘텐츠 생성, 분석) | 로컬 서버에서 실행 |

### 핵심 원칙

> **당신이 매일 해야 할 일은 "제안 승인/거부"뿐입니다.**  
> 나머지(시장 조사, 리드 수집, 콘텐츠 초안 생성, 고객 여정 관리)는 전부 AI가 자동으로 수행합니다.

## 2. 초기 세팅

### 2.1 필수 준비물

| 항목 | 용도 | 발급처 |
|------|------|--------|
| Naver API 키 | 뉴스/블로그/카페 검색 (시장 조사) | [developers.naver.com](https://developers.naver.com) |
| YouTube API 키 | 영상/댓글 수집 | [console.cloud.google.com](https://console.cloud.google.com) |
| Discord 웹후크 | 에러/상태 알림 | Discord 서버 채널 설정 |
| Instagram 계정 (선택) | 자동 팔로우/DM | 비즈니스 전용 계정 권장 |

### 2.2 Naver API 키 발급

1. [developers.naver.com](https://developers.naver.com) 접속 → 네이버 로그인
2. 상단 **"Application"** → **"애플리케이션 등록"**
3. 입력 사항:
   - 애플리케이션 이름: `WEET Director`
   - 사용 API: **검색** 체크
   - 비로그인 오픈 API 서비스 환경: **WEB 설정** → `http://localhost`
4. **등록** 클릭
5. 발급된 **Client ID**와 **Client Secret**을 복사해 보관

### 2.3 YouTube API 키 발급

1. [console.cloud.google.com](https://console.cloud.google.com) 접속 → Google 로그인
2. 상단 프로젝트 선택 → **"새 프로젝트"** → 이름: `WEET Director` → 만들기
3. 좌측 메뉴 **"API 및 서비스"** → **"라이브러리"**
4. `YouTube Data API v3` 검색 → 클릭 → **"사용"** 버튼
5. 좌측 **"사용자 인증 정보"** → **"+ 사용자 인증 정보 만들기"** → **"API 키"**
6. 생성된 API 키를 복사해 보관

### 2.4 Discord 웹후크 생성

1. Discord 서버에서 알림을 받을 채널 선택
2. 채널명 우클릭 → **"채널 편집"** → **"연동"** 탭
3. **"웹후크"** 섹션 → **"새 웹후크"** → 이름: `WEET Director`
4. **"웹후크 URL 복사"** 클릭 → 보관

```bash
python -m app.orchestrator.main --once

# 6. 상시 실행 (매일 자동 스케줄)
python -m app.orchestrator.main
```

## 3. 대시보드 사용법

### 접속 방법

1. `https://erp.we-et.com` 접속
2. 계정으로 로그인
3. 좌측 사이드바에서 **"마케팅"** 클릭
4. 6개 서브 메뉴 사용

```
마케팅
├── 개요        ← 전체 현황 한 눈에
├── 리드        ← 수집된 잠재 고객
├── 제안        ← AI 마케팅 액션 제안
├── 콘텐츠      ← 발행된 콘텐츠
├── 신호        ← 시장 동향/트렌드
└── 시스템      ← 인프라 상태
```

### 3.1 개요 페이지

> **경로**: `/marketing`  
> **역할**: 마케팅 현황을 한 눈에 파악

#### 화면 구성

| 영역 | 위치 | 설명 |
|------|------|------|
| **메트릭 카드** | 상단 4열 | 총 리드 / 대기 중 제안 / 발행된 콘텐츠 / 활성 채널 |
| **일일 추이 차트** | 중단 좌측 | 최근 14일간 바 차트 |
| **리드 파이프라인** | 중단 우측 | 6단계 고객 여정 분포 |
| **최근 활동** | 하단 전체 | 리드 수집, 제안 승인/거부, 콘텐츠 발행 이력 |
| **최근 제안** | 하단 좌측 | 대기 중인 제안 3건 (승인/거부 가능) |
| **최근 리드** | 하단 우측 | 최근 수집된 리드 5건 |

#### 메트릭 카드 읽는 법

각 카드에 `↑ +3 전주 대비` 또는 `↓ -2 전주 대비` 표시가 됩니다.
- **↑ 초록색**: 지난주보다 증가 (좋은 신호)
- **↓ 빨간색**: 지난주보다 감소 (주의 필요)
- **→ 변동없음**: 회색 (안정적)

이 숫자는 "이번 주(최근 7일) - 지난 주(7~14일 전)" 차이입니다.

#### 일일 추이 차트

상단 탭 3개를 클릭하면 다른 메트릭을 볼 수 있습니다:
- **리드 수집**: 매일 수집된 잠재 고객 수
- **제안 생성**: AI가 생성한 마케팅 제안 수
- **콘텐츠 발행**: 발행된 콘텐츠 수

#### 리드 파이프라인

고객 여정을 6단계로 시각화합니다:

```
인지 ████████████████ 42명
관심 ██████████       25명
고려 ██████           15명
결정 ████              8명
전환 ██                4명
유지 █                 2명
```

각 단계의 리드 수가 가로 막대로 표시됩니다. 파이프라인이 위에서 아래로 좁아지는 것이 정상입니다.

#### 여기서 할 수 있는 일

- **제안 승인**: 최근 제안 카드의 `승인` 버튼 → 즉시 처리
- **제안 거부**: `거부` 버튼 → "관리자 거부" 사유로 즉시 처리
- **전체보기 이동**: "전체보기" 링크 → 각 상세 페이지로 이동
- **차트 탭 전환**: 리드 수집 / 제안 생성 / 콘텐츠 발행

| 점수 | 의미 | 수집 소스 |
|------|------|----------|
| **8점** | 매우 높은 관심 | 경쟁사 게시물에 댓글 단 사용자 |
| **7점** | 높은 관심 | 네이버 카페에서 관련 질문한 사용자 |
| **6점** | 관심 있음 | 관련 해시태그를 사용하는 사용자 |
| **5점** | 잠재 관심 | 경쟁사 게시물에 좋아요 누른 사용자 |
| **4점** | 가벼운 관심 | YouTube 관련 영상에 댓글 단 사용자 |
| **3점** | 업계 관심 | 경쟁사 팔로워 |

#### 여기서 할 수 있는 일

- **검색**: 검색창에 사용자명이나 플랫폼 입력 → 0.3초 후 자동 필터링

### 3.3 제안 검토

> **경로**: `/marketing/proposals`  
> **역할**: AI가 생성한 마케팅 액션 제안을 검토하고 승인/거부

#### 이 페이지가 가장 중요합니다

스케줄러가 시장 신호를 분석해서 자동으로 마케팅 액션을 제안합니다. 담당자는 이 제안을 검토하고 승인하거나 거부하면 됩니다.

#### 화면 구성

| 영역 | 설명 |
|------|------|
| **상태 탭** | 전체 / 대기중 / 승인됨 / 거부됨 |
| **제안 카드 그리드** | 3열 그리드로 제안 카드 표시 |

#### 제안 카드 구성

```
┌─────────────────────────────────────┐
│ [액션 타입]  [대기중]        [HIGH] │
│                                     │
│ 제안 제목                           │
│                                     │
│ ┌─────────────────────────────────┐ │
│ │ AI가 생성한 콘텐츠 초안이       │ │
│ │ 여기에 표시됩니다...            │ │
│ └─────────────────────────────────┘ │
│                                     │
│    [ ✓ 승인 ]     [ ✕ 거부 ]       │
└─────────────────────────────────────┘
```

#### 승인/거부 워크플로우

**승인하는 경우:**
1. 제안 카드 내용을 읽습니다
2. `승인` 버튼 클릭
3. "제안이 승인되었습니다" 토스트 메시지 확인
4. 제안 상태가 `대기중` → `승인됨`으로 변경됩니다
5. 승인된 제안은 다음 스케줄러 실행 시 콘텐츠로 생성됩니다

**거부하는 경우:**
1. `거부` 버튼 클릭
2. 거부 사유 입력 모달이 표시됩니다
3. 거부 사유를 입력합니다 (선택 — 비워두면 "관리자 거부"로 기록)
4. `거부 확인` 버튼 클릭
5. 제안 상태가 `대기중` → `거부됨`으로 변경됩니다

#### 상태별 의미

| 상태 | 배지 색상 | 의미 |
|------|----------|------|
| **대기중** | 노란색 | 아직 검토되지 않음 — 확인 필요 |
| **승인됨** | 초록색 | 승인 완료 — 콘텐츠 생성 대기 |
| **거부됨** | 빨간색 | 거부됨 — 사유와 함께 기록 보관 |

### 3.4 콘텐츠 관리

> **경로**: `/marketing/content`  
> **역할**: 다양한 채널에 발행된 마케팅 콘텐츠 관리

#### 화면 구성

| 영역 | 설명 |
|------|------|
| **채널 탭** | 전체 / 블로그 / 인스타그램 / 카페 / 유튜브 / 당근 / 카카오 (각 채널별 개수 표시) |
| **상태 배지** | 현재 필터 기준 초안/검토/승인/발행/보관 건수 |
| **CSV 버튼** | 필터된 콘텐츠를 CSV로 다운로드 |
| **콘텐츠 카드** | 2열 그리드로 콘텐츠 프리뷰 표시 |

#### 콘텐츠 상태

| 상태 | 의미 |
|------|------|
| **초안** | AI가 생성한 초안 상태 |
| **검토 중** | 담당자가 검토 중 |
| **승인** | 발행 승인됨 |
| **발행됨** | 채널에 실제 발행 완료 |
| **보관** | 더 이상 활성화하지 않음 |

#### 지원 채널

| 채널 | 생성되는 콘텐츠 |
|------|-----------------|

| **인스타그램** | 캡션 + 해시태그 |
| **카페** | 네이버 카페 게시글 |
| **유튜브** | 영상 스크립트 |
| **당근** | 당근마켓 상품/홍보 글 |

#### 여기서 할 수 있는 일

- **CSV 내보내기**: `CSV` 버튼 → 채널/제목/상태/생성일/발행일 다운로드

### 3.5 시장 신호

> **경로**: `/marketing/signals`  
> **역할**: Market Radar가 감지한 업계 동향, 경쟁사 움직임, 트렌드

#### 화면 구성

| 영역 | 위치 | 설명 |
|------|------|------|
| **긴급도 탭** | 상단 | 전체 / 긴급 / 높음 / 보통 / 낮음 |
| **신호 타임라인** | 좌측 (2/3) | 시간순 신호 목록 |
| **트렌드 키워드** | 우측 상단 | 빈도 높은 키워드 Top 10 |
| **신호 분석 요약** | 우측 중단 | 총 건수, 긴급/높음 건수 |
| **필터 결과** | 우측 하단 | 현재 필터된 건수 |

#### 신호 타임라인 카드

각 신호는 타임라인 형태로 표시됩니다:

```
● [CRITICAL]  네이버 뉴스                      03/07 09:30
  ────────────────────────────────────────
  정부 소상공인 지원금 확대 발표
  
  ┌─────────────────────────────────────┐
  │ 중소기업부가 소상공인 대상 마케팅   │
  │ 지원금을 기존 100만원에서 200만원   │
  │ 으로 확대한다고 발표...             │
  └─────────────────────────────────────┘
  [소상공인] [지원금] [마케팅] [정부정책]
```

- **긴급도 배지**: 빨간색(CRITICAL) / 노란색(HIGH) / 회색(MEDIUM, LOW)
- **출처**: 네이버 뉴스, 블로그, 카페, YouTube 등
- **키워드 배지**: 신호에서 추출된 핵심 키워드 (최대 5개)

#### 트렌드 키워드

우측 사이드바에 전체 신호에서 가장 자주 등장하는 키워드 10개가 표시됩니다.
- 🔥 아이콘이 붙은 것은 **Top 3** 키워드입니다
- 이 키워드들의 변화를 주간 단위로 관찰하면 시장 트렌드를 파악할 수 있습니다

#### 여기서 할 수 있는 일

- **긴급도 필터**: 탭으로 긴급도별 필터링
- **트렌드 파악**: 우측 키워드 변화 모니터링

## 4. Python 스케줄러 운영

### 4.1 자동 실행 일정 (KST 기준)

스케줄러를 상시 실행하면 아래 시간표대로 자동 동작합니다:

| 시간 | 작업 | 설명 |
|------|------|------|
| 06:00 | `daily_reset` | 일일 상태 초기화 |
| **07:00** | **`market_scan`** | 네이버/유튜브 스캔 → 시장 신호 수집 |
| **08:00** | **`suggestion_run`** | 수집된 신호 분석 → 마케팅 제안 자동 생성 |
| 09:00 | `daily_report` | 일일 통계 집계 |
| 10:00 | `journey_check` | 고객 여정 트리거 확인 (자동 DM 등) |
| **13:00** | **`content_generate`** | 승인된 제안 → 콘텐츠 초안 생성 |
| 15:00 | `content_publish` | 생성된 콘텐츠 채널 발행 |
| **18:00** | **`lead_hunt`** | 잠재 고객 자동 수집 |
| 21:00 | `evening_followup` | 저녁 팔로우업 액션 |
| 월 09:00 | `weekly_report` | 주간 분석 보고서 |
| 1일 09:00 | `monthly_analysis` | 월간 심층 분석 |

> **굵게 표시된 4개 작업**이 핵심입니다. 이 작업들이 대시보드에 데이터를 공급합니다.

### 4.2 실행 명령어

```bash
# 가상환경 활성화
cd weet-director/backend
source .venv/bin/activate

# 실행 모드 3가지:
python -m app.orchestrator.main --dry-run   # 테스트 (API 호출 없음)
python -m app.orchestrator.main --once      # 한 번만 실행
python -m app.orchestrator.main             # 상시 실행 (스케줄 따름)
```

### 4.3 백그라운드 상시 실행

스케줄러를 끄지 않고 백그라운드에서 계속 돌리는 방법:

**방법 1: tmux (권장)**
```bash
# 새 세션 생성
tmux new -s weet-director

# 스케줄러 실행
cd weet-director/backend
source .venv/bin/activate
python -m app.orchestrator.main

# 분리 (백그라운드 유지): Ctrl+B 누른 후 D
# 다시 접속: tmux attach -t weet-director
# 종료: tmux kill-session -t weet-director
```

**방법 2: nohup**
```bash
cd weet-director/backend
source .venv/bin/activate
nohup python -m app.orchestrator.main > scheduler.log 2>&1 &

# 로그 확인
tail -f scheduler.log

# 종료
kill $(pgrep -f "app.orchestrator.main")
```

### 4.4 Ollama 관리

```bash
# Ollama 서버 시작 (스케줄러 실행 전 필수)
ollama serve

# 설치된 모델 확인
ollama list

# 현재 사용 모델:
# qwen3.5:35b   — 콘텐츠 생성 (primary + quality)
# qwen3.5:9b    — 빠른 분류/요약 (fast)
# qwen3-embedding:8b — 의미 분석 (embedding)
```

### 4.5 스케줄러 제한 설정

`.env` 파일에서 Instagram 자동화 일일 한도를 설정할 수 있습니다:

| 설정 | 기본값 | 설명 |
|------|--------|------|
| `DAILY_LIKES_LIMIT` | 150 | 하루 최대 좋아요 수 |
| `DAILY_FOLLOWS_LIMIT` | 50 | 하루 최대 팔로우 수 |
| `DAILY_COMMENTS_LIMIT` | 30 | 하루 최대 댓글 수 |
| `DAILY_DMS_LIMIT` | 15 | 하루 최대 DM 수 |
| `OPERATING_HOURS_START` | 7 | 활동 시작 시간 (KST) |
| `OPERATING_HOURS_END` | 23 | 활동 종료 시간 (KST) |

> ⚠️ Instagram 한도를 너무 높게 설정하면 계정 제한이 걸릴 수 있습니다. 기본값을 유지하는 것을 권장합니다.

### 전체 데이터 흐름

```
[07:00 Market Scan]
  │

  │
[08:00 Suggestion Run]
  │
  │  Ollama AI가 신호 분석 → 마케팅 액션 제안 자동 생성
  │
  ▼
[marketing_proposals 테이블] ──→ 대시보드 "/marketing/proposals"
  │
  │           ┌──────────────────────────┐
  │           │  ★ 담당자가 승인/거부 ★  │
  │           └──────────────────────────┘
  │
[13:00 Content Generate]
  │
  │  승인된 제안 기반 → 6개 채널 콘텐츠 초안 생성
  │
  ▼

  │
  │

  │

  │
  ▼

  │
  │
[10:00 Journey Check]
  │
  │  고객 상태 확인 → 자동 DM/리마인더 발송
  │
  ▼
[marketing_lead_actions 테이블]
```

### 고객 여정 자동 트리거

스케줄러가 자동으로 감지하고 액션을 수행합니다:

| 트리거 | 조건 | 자동 액션 |
|--------|------|----------|

| `profile_visit` | 프로필 방문 | 정보 안내 DM |
| `silence_3day` | 3일 무반응 | 리마인더 DM |
| `silence_7day` | 7일 무반응 | 사례 연구/후기 DM |
| `handoff_signal` | 구매 의향 키워드 감지 | 영업팀에 알림 |

## 7. FAQ / 문제 해결

### Q: 대시보드에 데이터가 안 보여요

**원인**: Python 스케줄러가 실행되지 않고 있습니다.

**해결**:
1. `/marketing/system` 페이지에서 스케줄러 상태 확인
2. 스케줄러가 "중지됨"이면:
   ```bash
   cd weet-director/backend
   source .venv/bin/activate
   python -m app.orchestrator.main --once  # 한 번 실행
   ```
3. 대시보드 새로고침

### Q: "데이터를 불러오지 못했습니다" 에러가 표시돼요

**원인**: API 서버 연결 문제 또는 인증 만료

**해결**:
1. 에러 메시지 아래 `다시 시도` 버튼 클릭
2. 반복되면 브라우저 새로고침 (F5)
3. 계속되면 다시 로그인

### Q: 제안이 자동으로 안 생겨요

**원인**: 시장 신호가 없거나 스케줄러가 중지됨

**해결**:
1. `/marketing/signals` 에서 최근 신호가 있는지 확인
2. 신호가 없으면 → 스케줄러의 `market_scan` 작업이 실행됐는지 확인
3. Naver API 키가 유효한지 확인 (`.env` 파일)
4. 스케줄러 로그에서 에러 확인:
   ```bash
   # tmux 방식
   tmux attach -t weet-director
   
   # nohup 방식
   tail -100 scheduler.log
   ```

### Q: Ollama 연결이 안 돼요

**해결**:
```bash
# 1. Ollama가 실행 중인지 확인
curl http://localhost:11434/api/tags

# 2. 실행 안 되면 시작
ollama serve

# 3. 필요한 모델이 있는지 확인
ollama list
# qwen3.5:35b, qwen3.5:9b, qwen3-embedding:8b 필요
```

### Q: Discord 알림이 안 와요

**해결**:
1. `.env`의 `DISCORD__WEBHOOK_URL`이 올바른지 확인
2. 웹후크 URL을 브라우저에 붙여넣어서 접근 가능한지 테스트
3. Discord 채널에서 웹후크가 삭제되지 않았는지 확인

### Q: CSV 파일이 엑셀에서 한글이 깨져요

**원인**: 인코딩 문제

**해결**: CSV 파일은 UTF-8 BOM 형식으로 생성됩니다. 대부분의 최신 엑셀에서 자동 인식하지만, 깨지는 경우:
1. 엑셀 → 데이터 → 텍스트/CSV에서 가져오기
2. 인코딩을 **UTF-8**로 선택
3. 구분 기호를 **쉼표**로 선택

### Q: 스케줄러를 완전히 중지하려면?

```bash
# tmux 방식
tmux kill-session -t weet-director

# nohup 방식
kill $(pgrep -f "app.orchestrator.main")

# 확인
ps aux | grep orchestrator
```
